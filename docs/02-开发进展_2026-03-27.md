## 6. 通讯与串口打印增强（本轮追加）

### 6.1 已完成增强

1. **临时协议补齐序号与校验**（仍保持 8 字节）
   - `byte4`: `seq`（发送序号，循环 0~255）
   - `byte6`: `checksum`（XOR 校验）
   - `byte7`: 固定标签 `0x5A`
   - 作用：便于快速判断丢包、重复包、误码包。

2. **RX 侧新增链路质量判定**
   - 统计项：`ok / parse_err / dup / jump`
   - 检测项：
     - 包解析失败打印 `[RX-ERR] bad packet`
     - 序号跳变打印 `[RX-WARN] seq_jump ...`
     - 超时（>1200ms 无有效包）打印 `[RX-WARN] timeout ...`

3. **TX 侧发送结果统计**
   - 发送成功/失败分别计数：`tx_ok / tx_fail`
   - 失败时即时打印 `[TX-ERR] send fail`

4. **串口日志可观测性增强**
   - TX 模式变化日志增加 `seq`
   - RX 模式变化日志增加 `seq`
   - 每 3 秒周期打印统计，避免刷屏：
     - TX: `[TX-STAT] ok=... fail=... seq=...`
     - RX: `[RX-STAT] ok=... parse_err=... dup=... jump=...`

### 6.2 当前收益

- 可以用**纯串口日志**快速定位问题属于：
  - 发不出去（TX 失败）
  - 收到但格式错误（parse_err）
  - 重复包（dup）
  - 丢包/乱序趋势（jump）
  - 链路中断（timeout）
- 为下一步“临时协议 → 正式业务协议（带设备 ID/按键事件）”提供了可复用骨架。
