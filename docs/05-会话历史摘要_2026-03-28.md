2. **对码指示节奏调整**
   - 对码进行中 LED 快闪周期调整为 `200ms`

4. **LED 优先级阶段性治理**
   - 对码窗口期间屏蔽普通短亮提示，避免抢占对码灯效

5. **RX 自动对码窗口**
   - RX 上电/复位自动进入 `5s` 对码窗口

6. **对码反馈灯效统一**
   - 成功：闪 `3` 次
   - 失败/超时：长亮 `1s`

7. **TX 按键提示灯**
   - 每次按键生效时 LED 闪一下（约 `100ms`）

## 5. 今日补充结论（按键专项）

1. 已完成右键本地调试：
   - 引入单击/双击/长按可视化 LED 回放；
   - 关闭 RF 干扰后验证，双击可偶发命中，但稳定性不足。
2. 现象归纳：
   - 在连续操作场景下，双击与单击/长按边界重叠明显；
   - 串口调试输出存在阻塞效应，影响实时性观感与节奏判断。
3. 决策（阶段性）：
   - 当前版本先收敛“单击 + 长按”链路；
   - 双击识别保留为后续专项优化项，不阻塞主功能推进。

## 6. 2026-03-29 追加改动（TX 发包与对码入口）

1. TX 发包从“固定周期持续发送”改为“按键事件触发短突发发送”（减少链路占用与 RX 漏收）。
2. TX 仅在进入对码窗口时维持低频周期发包；普通待机不再持续广播。
3. TX 上电不自动进入对码；仅保留 L+R 长按进入对码窗口。
4. 双击业务继续阶段性暂缓（单击 + 长按继续联调推进）。

## 7. 2026-03-29 追加改动（按键与RF热身）

1. 按键策略收敛为“单击 + 长按”即时触发：
   - 单击：去抖后松开立即判定；
   - 长按：按住达到阈值即触发，不必等待松手；
   - 双击逻辑关闭（仅保留参数与后续入口）。
2. TX 启动“保温窗口”策略：
   - 上电保持唤醒 `1500ms`，避免刚上电第一下按键不易触发；
   - 首次事件突发包增强（6 包），后续事件 4 包。
3. RF 睡眠/唤醒机制稳定：
   - 空闲进入休眠；
   - 事件唤醒后发送，完成后再次休眠。
