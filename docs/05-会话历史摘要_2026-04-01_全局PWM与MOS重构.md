## 3. 程序改动（已完成）

目标文件：`reference STM32c011f6p6-xl2400t/Core/Src/main.c`

### 3.1 IO/执行架构对齐

- 对齐最新引脚定义：
  - `PA2` = `PWM_GLOBAL`（TIM1_CH3）
  - `PA3` = `MOS_L_HI`
  - `PA6` = `MOS_L_LO`
  - `PA8` = `MOS_R_HI`
  - `PA10` = `MOS_R_LO`
  - `PB6` = `LED`（状态指示）
- 移除旧的双 PWM 启动路径（`TIM_CHANNEL_4` 不再参与灯效输出）。

### 3.2 新增/重构关键函数

1. `PwmGlobal_Set(uint16_t pulse)`
   - 职责：统一设置全局 PWM 占空比（只操作 `TIM1_CH3`）。

2. `Channel_ApplyMask(uint8_t mask)`
   - 职责：统一下发四路 MOS 导通位图。

3. `Light_Tick(uint32_t now)`
   - 重构为“模式 -> MOS 组合 + PWM 占空”执行模型：
     - mode1：近光（`L_LO + R_LO`）
     - mode2：远光（`L_HI + R_HI`）
     - mode3：同亮（四路导通）
     - mode4：远光同步快闪
     - mode5：A/B 交替（`L_HI+R_LO` / `R_HI+L_LO`）
     - mode6：近光呼吸（PWM 占空比变化）
     - mode7：远光左右轮闪（`L_HI` / `R_HI` 交替）

### 3.3 新增调试开关（便于台架排查）

- 宏：`PWM_GLOBAL_FORCE_LEVEL`
  - `0`：正常输出
  - `1`：强制低电平（0%）
  - `2`：强制高电平（100%）
- 目的：快速验证“PWM 与 MOS 是否独立且逻辑正确”。

- 统一强调边界：`PWM_GLOBAL` 负责亮度，`MOS` 负责通道选择。

## 6. 追加记录（2026-04-02）：按键响应卡顿优化（v2.10.8-KeyLatencyFix-RfTxPoll-UartBudget）

### 6.1 背景现象

- 现场反馈：按键偶发“已按下但未触发功能”；
- 已排除“仅由串口打印导致阻塞”的单一原因后，问题仍存在；
- 实测主观手感在优化后明显改善。

### 6.2 根因分析（本轮确认）

1. `RF_TX_Data()` 发送后固定 `RF_DelayMs(100)`，会在主循环形成约 100ms 的硬阻塞。
2. TX 三键识别虽然按 1ms 节拍推进，但该节拍由主循环驱动；主循环被阻塞时，按键状态机会“断采样”。
3. `UartCmd_Poll()` 原先为“吃空式 while 轮询”，若串口输入连续到来，会长时间占用主循环时间片。

结论：本问题属于**主循环实时性不足**，不是单点日志开关问题。

### 6.3 代码落地（已完成）

1. RF 发送后等待策略改为“可切换双模式”（宏开关）：
   - 固定延时模式（兼容旧行为）；
   - 快速轮询模式（默认，短超时轮询 `TX_DS/MAX_RT`）。
2. 串口命令轮询增加“单轮字节预算 + 回显开关”：
   - 每轮最多处理 `UART_CMD_POLL_MAX_BYTES` 个字节；
   - `UART_CMD_ECHO_ENABLE` 控制是否字符回显。
3. 固件版本更新：`FW_VERSION` 升级为 `v2.10.8-KeyLatencyFix-RfTxPoll-UartBudget`。

### 6.4 本轮收益

- 按键处理的实时性和一致性明显改善；
- 双击/短按判定窗口更稳定，漏触发概率降低；
- 保留宏开关后，便于台架 A/B 对比测试与回归。

### 6.5 对比测试建议（A/B）

1. A 组（旧行为）：`RF_TX_USE_FIXED_DELAY_MODE=1`，`RF_TX_POST_CE_FIXED_DELAY_MS=100`。
2. B 组（新行为）：`RF_TX_USE_FIXED_DELAY_MODE=0`，`RF_TX_STATUS_WAIT_TIMEOUT_MS=3`。
3. 在两组下分别进行：单击 100 次、双击 100 次、长按 50 次，记录漏触发率与误判率。

### 6.6 追加进展（2026-04-02）：TX 低功耗 Step1（v2.10.9-TxStopLowPower-Step1）

根据低功耗排查目标，先落地“可切换 + 可回退”的第一步：

1. 新增 TX 低功耗模式宏：
   - `TX_LOWPOWER_MODE_SLEEP`
   - `TX_LOWPOWER_MODE_STOP`
   - `TX_LOWPOWER_MODE`（当前默认 `STOP`）
2. 进入低功耗前统一收敛（无条件执行）：
   - 下发 `RF_Link_Sleep()`；
   - 固定 RF 三线电平：`CSN=高`、`SCK=低`、`DATA=低`；
   - 关闭串口（`HAL_UART_DeInit`）。
3. 低功耗入口：
   - STOP 模式：`HAL_PWR_EnterSTOPMode(PWR_LOWPOWERREGULATOR_ON, WFI)`；
   - SLEEP 模式：保留旧路径（便于 A/B 对比）。
4. 唤醒恢复：
   - STOP 唤醒后先 `SystemClock_Config()`，再 `MX_USART1_UART_Init()`；
   - SLEEP 唤醒后直接 `MX_USART1_UART_Init()`。

说明：本步按“快速唤醒优先”原则实现，RF 下发睡眠后不增加额外等待延时，便于按键快速唤醒测试。

### 6.7 追加进展（2026-04-02）：低功耗与业务主流程收敛（v2.10.13 ~ v2.10.17）

#### A. 关键里程碑

1. `v2.10.13-CfgBanner-MacroCentral`
   - 将角色/链路测试/低功耗/诊断等关键宏集中到 `main.c` 前部配置区；
   - 启动新增 `[CFG] role=... link_test=... lowpower=...` 打印，避免烧录身份混淆。

2. `v2.10.14-RoleFlowUnify-LP`
   - 业务收发主流程从 `APP_RF_LINK_TEST` 分支中解耦，恢复“正常模式也执行完整收发”；
   - `APP_RF_LINK_TEST` 收敛为“附加诊断开关”，不再决定是否跑业务。

3. `v2.10.15-LPWakeGuard-KeyCapture`
   - 修复 TX 低功耗唤醒后的 RF 状态恢复：业务繁忙时强制 `RF_Link_ConfigTx()`，避免“唤醒了但没发包”。

4. `v2.10.16-LPWakeGuard-KeyHold`
   - 新增“唤醒保护窗口 + 按键活动保持窗口”，抑制 STOP 后快速回睡导致的漏按。

5. `v2.10.17-LPNoSleepWhilePress`
   - 增强为“按键按下期间禁止进入低功耗”；
   - 将窗口参数提升为 `TX_LP_WAKE_GUARD_MS=1000ms`、`TX_LP_KEY_ACTIVE_HOLD_MS=1400ms`，提升睡眠后长按稳定性。

#### B. 本日测试结论（阶段性）
