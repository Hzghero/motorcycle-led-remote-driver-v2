## 2. 产品概述

本产品为可遥控的摩托车远近光 LED 驱动器，用于驱动双灯（远光、近光）射灯系统。远光和近光均为 30W，可通过 2.4G 无线遥控器（STM32C011F6P6TR + XL2400T 模块）或原车灯光/喇叭信号进行控制。两灯支持多种交互灯光效果，包括呼吸灯、同步闪、交替闪等。

**主要特性**：
- 远光 30W + 近光 30W 双路独立控制
- 2.4G 无线遥控（发射端 STM32C011F6P6TR + XL2400T（TX only），接收端 STM32C011F6P6TR + XL2400T（RX only）），支持对码配对
- 三档亮度调节（30% / 70% / 100%）
- 七种灯光模式（含呼吸灯、同步闪、交替闪等）
- 可与原车远光、喇叭联动
- 适用于摩托车 12V 电气系统

**主要模块**：
- **主控**：STM32C011F6P6TR（20-TSSOP，18 I/O）
- **2.4G 接收**：XL2400T（接收端单向接收，与主控通过 RF_CSN/RF_SCK/RF_DATA 三 IO 通讯）
- **LED 驱动**：LM3409 × 2，主控 2 个 GPIO 输出 PWM 分别控制
- **车辆接口**：电源、ACC、黄线（远光）、蓝线（喇叭）、天线

### 4.2 LED 恒流驱动（LM3409）

- **芯片**：LM3409，每路灯独立一颗
- **PWM 与开关复用**：同一路控制信号
  - 恒高电平 → 100% 电流（全亮）
  - 恒低电平 → 关断（熄灯）
  - PWM 占空比 → 对应百分比的电流（亮度）
- **主控输出**：2 个 GPIO 输出 PWM，分别接到两颗 LM3409
- **实现方式**：所有灯光效果（常亮、呼吸灯、同步闪、交替闪等）均由主控调节这两路 PWM 的占空比和时序实现

## 5. 无线遥控

### 5.1 遥控器结构

- **三按键设计**
- **左键（锁图标）**：基础灯光模式切换
- **中键**：亮度调节，每次按下循环切换 30% → 70% → 100%
- **右键（电源/波形图标）**：特效模式切换
- 内置指示灯（对码时红色常亮）
- 电池：CR2032 纽扣电池，约 1 年寿命

### 5.2 亮度调节

- 中键按下时，亮度在 **30% / 70% / 100%** 三档循环
- 亮度通过调节恒流驱动的 PWM 占空比实现
- 当前亮度档位对所有灯光模式均有效（常亮、呼吸灯、闪烁等均受其约束）
- PWM 约束（当前实现）：
  - **30% 档 / 70% 档**：输出 PWM 频率约 **20kHz**，占空比分别为 30% / 70%
  - **100% 档**：输出恒高电平（全亮），不再输出 20kHz PWM
- 在特效模式下，亮灭节奏（如 500ms 高电平 / 500ms 低电平）叠加在上述亮度驱动规则上：
  - 亮相位：按当前亮度档位输出（30/70 为 20kHz PWM，100 为恒高）
  - 灭相位：输出低电平（关断）

### 5.3 2.4G 射频方案（发射端 STM32 + XL2400T / 接收端 XL2400T）

| 项目 | 说明 |
|------|------|
| 芯片 | 发射端：STM32C011F6P6TR + XL2400T（TX only，遥控端）；接收端：STM32C011F6P6TR + XL2400T（RX only） |
| 通信方式 | **单向**：遥控器仅发射，接收器仅接收 |
| 遥控器 | TX only，无需接收功能 |
| 接收器 | RX only；与主控通过 RF_CSN/RF_SCK/RF_DATA 三 IO 通讯 |
| 频率 | 2.4GHz |
| 对码 | 支持一对一对码，更换遥控器或驱动器后可重新配对 |

**改动备注**：发射端无线方案由集成式 XL2417D 调整为“STM32C011F6P6TR + XL2400T（TX only）”，目的是复用 STM32 生态与已验证成功案例，固件与对码流程按接收端 XL2400T 的实现思路落地；发射端具体 GPIO/IO 引脚连接将在后续原理图阶段确定。

## 6. 灯光模式详细说明

**说明**：以下各模式的亮度均受当前亮度档位（30% / 70% / 100%）约束，由中键调节。

### 6.1 基础模式（左键控制）

| 操作 | 效果 |
|------|------|
| 单击 | 近光开 |
| 再单击 | 切换为远光 |
| 再单击 | 切换回近光（循环） |
| 双击或长按（约 1 秒） | 远近光同时亮（左键特例：首次触发同亮） |
| 再次双击或长按（约 1 秒） | 熄灯/点亮切换（支持任意键触发；左键在同亮后再次触发则熄灯） |

 
### 6.2 特效模式（右键控制）

| 操作 | 效果 |
|------|------|
| 第 1 次单击 | 远光同步快闪 |
| 第 2 次单击 | 远近光同步交替闪 |
| 第 3 次单击 | 近光同步呼吸灯 |
| 第 4 次单击 | 远光左右轮闪 |
| 再单击 | 循环上述特效 |

> 说明：从特效模式回到基础模式时，默认回到“近光开”。

### 6.3 熄灯

- 任意键（左/中/右）支持两种熄灯触发：
  - **双击**（快速连续击）
  - **长按约 1 秒**

> **阶段说明（2026-03-28 之后执行）**：
> 当前固件阶段先保留“单击 + 长按”的完整业务链路，
> “双击”识别存在误判风险，暂缓纳入主流程（保留为待优化项）。
- 优先级规则：
  - 左键双击/长按首次触发“远近同亮”；
  - 左键在“远近同亮”状态下再次双击/长按触发“熄灯”；
  - 中键/右键双击或长按一次即直接熄灯。
- 熄灯后再次点亮：
  - 触发方式为**任意键单击**；
  - 其中“单击右键”优先规则：**直接进入特效模式1**（远光同步快闪）；
  - 单击左键或单击中键：恢复为基础模式“近光开”；
  - 亮度档位保持熄灯前记忆值（中键档位记忆）。
- 进入特效模式后，按左键单击可退出特效并恢复基础模式（回到“近光开”）。
- 调试要求：熄灯/点亮动作需串口打印触发来源（按键 + 双击/长按/单击），用于联调核对。

> 阶段说明（2026-03-29 联调决议）：
> 当前固件联调阶段先收敛“单击 + 长按”链路，双击识别作为后续专项优化项，不阻塞主功能推进。

### 6.4 特效模式频率/时序参数表（可调）

> 说明：下表用于固件参数对齐与后续调参。当前为文档配置值，后续如需改动可直接改本表并同步代码。

| 特效模式 | 推荐时序定义 | 频率换算（参考） | 亮相位输出规则 |
|----------|--------------|------------------|----------------|
| 模式4 远光同步快闪 | 250ms 亮 + 250ms 灭（循环） | 2Hz | 30/70档=20kHz PWM 对应占空比；100档=恒高 |
| 模式5 远近交替闪 | 远光 500ms 亮（近光灭）→ 近光 500ms 亮（远光灭） | 每路 1Hz（交替） | 同上 |
| 模式6 近光呼吸 | 1 个完整呼吸周期（建议 2s，可调） | 0.5Hz 周期 | 呼吸相位内按亮度档位约束（30/70 为 PWM，100 为恒高） |
| 模式7 远光左右轮闪 | 左右通道轮流 250ms/250ms（可调） | 每路约 2Hz（轮闪） | 同上 |

**全局规则（亮度档位与PWM）**：
- 30% 档、70% 档：采用 **20kHz PWM** 调光，占空比分别为 30%/70%。
- 100% 档：采用**恒高电平**输出（非 20kHz PWM 占空调制）。
- 特效模式下“亮相位”同样遵循以上规则：亮相位期间按当前亮度档位输出，灭相位输出低电平。

### 6.5 模式汇总

| 序号 | 模式名称 | 远光 | 近光 | 说明 |
|------|----------|------|------|------|
| 1 | 近光 | 关 | 亮 | 常亮 |
| 2 | 远光 | 亮 | 关 | 常亮 |
| 3 | 远近同亮 | 亮 | 亮 | 两灯同时常亮 |
| 4 | 远光同步快闪 | 闪 | 关 | 远光快频率闪烁 |
| 5 | 远近交替闪 | 闪 | 闪 | 两灯交替闪烁 |
| 6 | 近光呼吸灯 | 关 | 呼吸 | 近光亮度渐变 |
| 7 | 远光左右轮闪 | 轮闪 | 关 | 左右灯轮流闪（若为双灯物理分离） |

## 8. 车辆集成

### 8.1 黄线（同步原车远近光）

- 黄线接原车**远光正极**后，可同步原车开关
- **有超车键**：连续拨 3 次超车键 → 激活射灯近光长亮
- **无超车键**：连续拨 3 次远近光开关 → 激活射灯近光长亮

### 8.2 蓝线（喇叭爆闪）

- 蓝线接喇叭正极
- 按喇叭时，射灯进入爆闪模式

### 8.3 白线（ACC 熄灯）

- 正确接好白线后，关钥匙门应能熄灯
- 若将白线接至大灯正极，可实现关原车灯时射灯同时熄灯

### 8.4 负控车型

- 若原车为负控（负触发），黄线、蓝线需接负控转换器，否则相关功能无法实现
- 电动车/电摩接线请另行确认

## 9. 对码流程

### 9.1 对码操作步骤

更换遥控器或驱动器后，需重新对码：

1. 断开车辆电源
2. 同时长按遥控器**左键 + 右键**约 5 秒（中键不参与对码）
3. 遥控器指示灯变红后松手
4. 接通车辆电源（或启动车辆）
5. 射灯远光闪 2～3 次后熄灭 → 对码成功

### 9.2 遥控器 ID 方案（首次随机 + 存储）

| 项目 | 说明 |
|------|------|
| 方案 | 首次随机生成 + 非易失存储 |
| 首次上电 | 遥控器 MCU 生成 32 bit 随机 ID，写入 Flash/EEPROM |
| 之后上电 | 从 Flash/EEPROM 读取已存储的 ID，不再重新生成 |
| 优点 | 无需工厂烧录；换电池后无需重新对码；碰撞概率极低（32 bit 空间约 43 亿） |
| 随机源 | 建议使用 ADC 噪声 + 定时器、或硬件 RNG（若有）作为种子 |

## 11. 时钟与定时基准（固件约束）

- MCU 时钟源：**外部 24MHz 晶振直连**（当前工程不使用 PLL 倍频）。
- 计时相关参数（如 TIM14 节拍、PWM 频率、延时估算）均以 **24MHz** 为基准。
- 若后续改动系统时钟，必须同步复核：
  - TIM1 PWM 频率与占空比映射；
  - TIM14（或其他基础定时器）节拍参数；
  - 与 `HAL_GetTick()` 相关的模式时序常量。

> 说明：该项为本项目关键硬件约束，调参或移植时优先检查。
