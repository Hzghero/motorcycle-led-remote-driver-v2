## 2. 关键改动摘要

## 2.1 引脚重映射（TX 侧）

- `L_KEY`: `PB7 -> PB6`（对应 `WKUP3`）
- `R_KEY`: `PA1 -> PA2`（对应 `WKUP4`）
- `PWM_GLOBAL`: `PA2/TIM1_CH3 -> PA1/TIM1_CH2`
- `LED`: `PB6 -> PB7`

## 2.2 STANDBY 三键唤醒补偿逻辑

- 启动阶段读取并打印 `WUF1/WUF3/WUF4`；
- 对应关系：
  - `WUF1 -> M_KEY`
  - `WUF3 -> L_KEY`
  - `WUF4 -> R_KEY`
- 若检测到唤醒来源，挂起一次对应按键“单击补偿”；
- 在按键扫描阶段对“仍按下”场景做过滤（跳过注入），避免“单击+长按”双触发。

## 2.3 低功耗手动上下拉兜底（关键）

由于新按键引脚在 Cube 配置上拉/下拉存在限制，已在 `EnterTxLowPower()` 手动补偿：

- `PA0(PD)`、`PA2(PD)`、`PB6(PD)`：按键线低功耗防悬空；
- `PA4(PU)`、`PA5(PD)`、`PA7(PD)`：RF 三线待机保持；
- 避免待机误唤醒与静态功耗抬升。

## 2.4 状态持久化与寿命保护（延续）

- 状态变化仅标记 dirty；
- 延迟合并保存（defer）；
- 最小写入间隔节流（min interval）；
- 进 STANDBY 前兜底保存（防止最终状态丢失）；
- 上电打印累计写入次数：`[TX-STATE] save_count=...`。

## 4. 回归验证结论（当前）

1. 三键唤醒路径可触发且有业务响应；
2. PWM 迁移后亮度控制链路正常；
3. 3 秒唤醒活跃窗口生效（不再 200~300ms 立刻回睡）；
4. 状态持久化不再每次按键都写，日志可观察节流效果。
