# 单向通信方案正式版（v1.0）

**文档版本**：v1.0  
**发布日期**：2026-03-27  
**适用范围**：本“新项目”遥控发射端（TX only）与灯控接收端（RX only）通信联调与量产前固化。  
**明确排除**：旧项目中的“日夜判断逻辑、充放电逻辑、相关合并回归项”全部不在本方案范围内。

---

## 1. 当前结论（基于已完成验证）

已完成“单端发射 + 另一端接收”联通验证，链路可稳定建立。

现阶段目标从“能通信”升级为“可量化、可维护、可交付”：
- 通信异常可被快速定位；
- 协议字段可冻结并长期演进；
- 串口日志可支持现场定位问题。

---

## 2. 系统边界与原则

### 2.1 系统边界
- 发射端：遥控器 MCU + XL2400T，**仅发送**。
- 接收端：灯控 MCU + XL2400T，**仅接收**。
- 业务边界：围绕遥控灯光控制，不引入旧项目电源管理逻辑。

### 2.2 设计原则
1. **坚持单向通信**：不引入 ACK 回传机制。  
2. **先稳后繁**：先冻结通信骨架，再逐步接入按键业务。  
3. **可观测优先**：必须能通过串口快速区分“发不出、收不到、包损坏、丢包/乱序”。
4. **参数持久化**：`device_id` 采用 Flash 持久化，掉电不丢失。

## 3. 协议正式化（v1.0）

> 说明：当前固件已使用 8 字节临时包完成联调，本版在该基础上正式命名字段并冻结语义。

### 3.1 帧格式（8 Byte）

| Byte | 字段 | 含义 |
|---|---|---|
| 0 | `header0` | 固定 `0xAA` |
| 1 | `header1` | 固定 `0x55` |
| 2 | `cmd_or_mode` | 当前阶段作为模式/命令字段 |
| 3 | `param_or_bright` | 当前阶段作为参数/亮度字段（0/1/2） |
| 4 | `seq` | 序号，0~255 循环 |
| 5 | `device_id` | 设备 ID（当前 8-bit；配对过滤） |
| 6 | `checksum` | XOR 校验 |
| 7 | `tag` | 固定 `0x5A` |

### 3.2 校验规则
`checksum = byte0 ^ byte1 ^ byte2 ^ byte3 ^ byte4 ^ byte5 ^ byte7`

### 3.3 RX 判定规则
接收端仅在满足以下条件时接收为有效包：
1. 帧头正确：`AA 55`
2. `tag == 0x5A`
3. `checksum` 校验通过
4. `device_id` 与已配对 ID 一致

否则记入 `parse_err` 或 `dev_miss` 并丢弃。

## 4. 串口日志正式规范（v1.0）

### 4.1 事件日志
- TX 发送失败：`[TX-ERR] send fail`
- RX 包错误：`[RX-ERR] bad packet`
- RX 序号跳变：`[RX-WARN] seq_jump prev=.. now=..`
- RX 超时：`[RX-WARN] timeout > 1200ms`
- 模式变化日志：
  - TX：`[TX] mode=.. bright=.. seq=.. dev=..`
  - RX：`[RX] mode=.. bright=.. seq=..`

### 4.2 周期统计日志（默认每 3s）
- TX：`[TX-STAT] ok=.. fail=.. seq=.. dev=..`
- RX：`[RX-STAT] ok=.. parse_err=.. dev_miss=.. dup=.. jump=..`

### 4.3 指标解释
- `ok`：有效包数量
- `parse_err`：格式/校验错误
- `dev_miss`：设备 ID 不匹配（非已配对遥控器）
- `dup`：重复包
- `jump`：序号跨越（疑似丢包或乱序）
- `timeout`：超时无有效包

---

## 5. 联调验收标准（通信专项）

### 5.1 稳态场景（近距离）
- 连续运行 30 分钟：
  - `TX fail = 0`
  - `RX parse_err = 0`
  - `RX dup = 0`
  - `RX jump` 接近 0（若不为 0，需提供时间点与现场说明）

### 5.2 干扰场景（中距离/有遮挡）
- 允许 `jump` 偶发，但不得持续高频增长；
- 不得出现系统卡死、异常复位、灯效失控。

### 5.3 恢复场景
- RX 断电重启后应在短时间内恢复接收；
- 恢复后统计继续可用。

- 按上述验收标准完成 30~60 分钟留档测试。
